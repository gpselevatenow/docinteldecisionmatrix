Data Sovereignty Note

Framework

Buy, Build, or Blend — Defined

Build

Open-source, self-hosted components within the carrier VPC. No vendor dependency, no data egress. Correct when determinism is required or posture constrains cloud usage.

Buy

Managed commercial services: AWS Textract, Azure Form Recognizer, Google Document AI. Fastest to production. Appropriate for Posture A where data sovereignty allows cloud processing.

Blend

Deterministic extraction layer (T0–T2 build) paired with an LLM reasoning layer (T4 local or T5 cloud). Correct architecture for any task requiring both structure and interpretation.

Data Posture Reference

A
Cloud-Safe

No PHI or regulated PII in scope. Cloud APIs are permissible. Standard commercial lines without medical data.

B
No PHI to Cloud

Documents contain PII, PHI, or carrier-sensitive data. External LLM APIs prohibited. Local/VPC-deployed models (T4) required.

C
Air-Gapped

Fully isolated environment. No network egress permitted. T0–T2 deterministic stack and local T4 models only.

Non-Chinese LLM Policy — Provider Reference

Carriers adopting a non-Chinese LLM policy exclude models from companies subject to China's National Intelligence Law (2017), which requires entities to cooperate with state intelligence operations. The following providers are aligned with this policy:

Anthropic Claude (US) OpenAI GPT-4 (US) Mistral (EU — France) Meta Llama (US) Google Gemini (US)

Excluded: Alibaba Qwen  ·  Baidu ERNIE  ·  DeepSeek  ·  Moonshot AI  ·  01.AI (Yi)  ·  Zhipu GLM
